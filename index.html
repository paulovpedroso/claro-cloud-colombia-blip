01 · Contexto y oportunidad

Colombia: 37,42 millones de usuarios y un líder de mercado.

Claro lidera el mercado colombiano de telecomunicaciones. Para Claro Cloud — la unidad B2B con Microsoft y AWS como principales fabricantes — esto representa un punto de partida ideal: alta densidad de potenciales clientes empresariales, embudo de venta con margen para optimización y un caso piloto que define la entrada de Blip a toda America Móvil.

⚠ El problema actual

Embudo de ventas con fricción real en el portal.

El sitio actual de Claro Cloud Colombia presenta el catálogo en lenguaje técnico (Microsoft 365, Azure, AWS Backup, etc.) sin guía consultiva. El usuario PyME llega, no entiende qué producto resuelve su problema, y abandona la página sin dejar rastro en el CRM.

  • Catálogo técnico sin traducción a casos de uso reales
  • Cero asistencia consultiva 24/7 en horario no comercial
  • Visitantes calificados que abandonan sin conversación
  • Equipo comercial reactivo, dependiente de formularios
✦ La propuesta

Dos agentes especializados, un solo Web Chat.

Un equipo conversacional embebido en el portal de Claro Cloud Colombia. El Agente Consultor aplica un framework consultivo para entender el problema del cliente y traducir SKUs técnicos a soluciones reales. El Agente SDR recomienda, hace cross-sell/up-sell y arma el carrito de compra dentro de la conversación.

  • Embebido en el portal · sin tocar el sitio actual
  • Español neutro · escalable a toda LATAM
  • Carrito de compras dentro del Web Chat
  • Handoff a Blip Desk + Copilot con contexto 360°
02 · Los dos agentes

Consultor primero.
SDR cierra.
Humano cuando hace sentido.

La diferencia entre un chatbot que responde preguntas y un agente conversacional que vende: el orden. Primero entendemos la necesidad, después recomendamos productos. Cada agente tiene un rol específico y un momento exacto de actuar.

v1 · Agente 01

Agente Consultor

// Diagnostica · Educa · Califica

Primer punto de contacto. Aplica un framework de venta consultiva para entender el contexto del cliente, su industria, su problema, su nivel técnico y su urgencia. Traduce el catálogo técnico de Claro Cloud a lenguaje de negocio. No vende — prepara el terreno para vender.

D
Descubrimiento conversacional — pregunta qué problema necesita resolver, no qué producto quiere comprar.
T
Traducción técnica — convierte SKUs como "Microsoft 365 E3" en "tu equipo trabajando con Word, Excel y Teams en cualquier dispositivo, con seguridad empresarial".
C
Calificación BANT consultiva — entiende presupuesto, autoridad, necesidad y timing sin formularios.
→
Handoff al Agente SDR — cuando el cliente está listo para evaluar opciones concretas, pasa el contexto completo.
v1 · Agente 02

Agente SDR

// Recomienda · Cross-sell · Up-sell · Cierra

Toma el bastón cuando el contexto está claro. Conoce todo el catálogo de Claro Cloud — SKUs, pricing, bundles ganadores, comparativos vs hyperscalers. Recomienda combinaciones inteligentes, hace cross-sell y up-sell, y arma el carrito de compras dentro del Web Chat antes de pasar a humano o cerrar directo.

R
Recomendación contextual — propone el SKU correcto según industria, tamaño y caso de uso identificado por el Consultor.
↗
Cross-sell y up-sell inteligente — el cliente que entra por Microsoft 365 sale evaluando Backup + Security; el de M365 Business pasa a M365 E3.
⊕
Carrito de compras integrado — arma el stack completo dentro del Web Chat, con totales y descuentos por bundle.
⌛
Cierre directo o handoff — operaciones simples cierran en línea; oportunidades complejas escalan a Blip Desk.

Handoff inteligente entre agentes: no es una transferencia visible para el cliente — es una continuidad de conversación. El Consultor pasa el contexto completo (industria, caso de uso, perfil técnico, BANT) al SDR. El cliente percibe un solo asistente que ahora conoce sus necesidades.

03 · Framework consultivo

El método de venta consultiva que entrena a ambos agentes.

Construido sobre metodologías probadas de venta consultiva B2B, adaptado al portafolio de Claro Cloud. Cinco etapas que el Agente Consultor recorre de forma natural — sin que el cliente sienta que está siendo entrevistado.

D
Descubrimiento

¿Qué problema de negocio estás tratando de resolver?

I
Industria

¿En qué sector operas y cuál es tu tamaño?

N
Nivel técnico

¿Cuán técnico es tu equipo? Adapta el lenguaje.

U
Urgencia

¿Cuándo necesitas tener esto funcionando?

A
Autoridad

¿Tomas la decisión o validas con alguien?

Ejemplo de aplicación · cliente PyME en retail

D · Descubrimiento

"¿Qué te trae a Claro Cloud hoy? ¿Hay algún desafío específico que estás tratando de resolver?" → Cliente responde: "Necesito que mi equipo trabaje desde casa con archivos compartidos."

I · Industria

"Genial, te ayudo. Para recomendarte bien: ¿en qué sector estás y cuántos colaboradores son?" → Cliente: "Retail, 25 personas en oficina y tienda."

N · Nivel técnico

"¿Tienen un equipo IT interno o tercerizan la administración?" → Cliente: "No, lo manejamos en la marcha." → El agente baja la complejidad técnica del lenguaje y prioriza soluciones gestionadas.

U · Urgencia

"¿Para cuándo lo necesitas operativo?" → Cliente: "Idealmente en 2-3 semanas." → El agente prioriza productos con activación rápida y onboarding asistido.

A · Autoridad

"¿Vos decidís directamente o validas con alguien más?" → Cliente: "Yo decido, pero coordino con mi socio." → El agente prepara material para compartir con el socio antes del cierre.

06 · Integraciones técnicas

Conectado al CRM.
Conectado al equipo.
Conectado al portal.

Tres integraciones críticas para que el agente opere sin fricción con la operación actual de Claro Cloud Colombia. Sin middleware custom — solo configuración vía Blip Studio.

⊕

Embebido en el portal

Web Chat instalado vía un único snippet de JavaScript en clarocloud.com.co. No requiere rebuild ni cambios en el sitio actual. Activación en horas, no semanas.

  • Snippet de instalación < 50 líneas
  • Configuración de triggers desde Blip Studio
  • Personalización visual con identidad Claro
  • A/B testing de copys nativo
⇄

CRM Claro Cloud Colombia

Cada lead calificado se escribe en tiempo real en el CRM local con owner asignado, score BANT, transcripción completa y producto de interés. Sin pérdida de contexto.

  • Mapeo de campos vía Blip Studio
  • Enriquecimiento con NIT (DIAN) automático
  • Routing por industria y tamaño
  • Attribution tracking end-to-end
✦

Blip Desk + Copilot

Cuando el agente hace handoff, el ejecutivo de Claro Cloud recibe el caso en Blip Desk con perfil completo. Copilot sugiere la próxima mejor acción y reduce el tiempo medio de atención.

  • Resumen automático de la conversación
  • Sugerencias en vivo basadas en casos similares
  • Score BANT y recomendación visible
  • Update automático del CRM al cerrar
